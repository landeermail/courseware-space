# AGENTS.md

本文件适用于整个仓库，供 Codex 和其他自动化开发代理执行任务时遵循。

## 项目边界

- 本项目是直接部署到 GitHub Pages 的静态站点，没有构建步骤或包管理器。
- 未经任务明确要求，不要引入 npm、前端框架、打包器或新的全站依赖。
- 优先做范围明确的修改；不要顺带重构无关课件。
- 保留各课件现有的资源组织方式。不要仅为统一目录而移动依赖、图片或 PWA 文件。

## 新增课件

- 将课件放在与主题对应的目录中，目录名使用小写英文和连字符。
- 每个课件目录必须包含可直接访问的 `index.html`。
- 图片、脚本、图标等专用资源应保存在课件目录内，避免依赖仓库根目录的相对路径。
- 同时更新根目录 `index.html` 中的 `coursewareData`，确保首页可以进入新课件。
- 首页登记路径使用相对路径，并以 `/` 结尾，例如 `electromagnetism/q20-rotating-rod/`。

## 实现要求

- 保持中文界面和物理术语准确；涉及公式、方向或单位时，不要凭视觉效果替代物理验证。
- 页面必须兼容常见桌面和移动视口。
- 交互不能只依赖鼠标悬停；3D 场景和主要控件应支持触控使用。
- 避免页面加载后自动播放无法停止的动画；持续动画应尊重页面可见性并避免不必要的性能消耗。
- 保持资源路径的大小写完全一致，因为 GitHub Pages 的文件路径区分大小写。
- 如果使用外部 CDN，应考虑资源不可用时页面的表现；未经要求不要替换现有课件的依赖来源。

## 验证

修改后，先在仓库根目录运行快速静态校验：

```bash
python3 scripts/validate_site.py
```

修改校验脚本或对应工作流时，另运行：

```bash
python3 -m unittest discover -s scripts -p "test_*.py"
```

涉及课件页面时，再启动本地服务器：

```bash
python3 -m http.server 8000
```

至少检查：

- 根目录首页能正常打开；
- 本次修改的课件页面能正常打开；
- 控制台没有由本次修改引入的错误；
- 图片、脚本和其他相对资源没有 404；
- 首页入口指向正确目录；
- 桌面和移动尺寸下，主要内容及交互可用。

纯文档修改可以只检查链接、命令和描述是否与仓库实际结构一致。

## Git 协作

- 开始任务前确认工作树状态，保留用户已有的未提交修改。
- 从最新 `main` 创建目的明确的分支，例如 `feat/...`、`fix/...` 或 `chore/...`。
- 不要提交临时服务器文件、系统文件或无关格式化改动。
- 完成并验证同一任务的全部关联改动后再集中提交，不要为每个小步骤单独提交或反复请求提交。
- `main` 上的提交会触发线上部署；正常变更应通过 Pull Request 合并。

## Agent skills

### Issue tracker

使用 GitHub Issues 记录重要开发任务；小型修正可以直接通过分支和 Pull Request 完成。详见 `docs/agents/issue-tracker.md`。

### Triage labels

当前未配置专门的任务分流工作流，不要求使用固定的 triage 标签。详见 `docs/agents/triage-labels.md`。

### Domain docs

项目采用单一领域上下文，由根目录 `CONTEXT.md` 记录共同知识，并在 `docs/adr/` 保存重要决策。详见 `docs/agents/domain.md`。
