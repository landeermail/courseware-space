# Issue 标签策略

本项目当前没有专门的任务分流（triage）工作流，也不配置固定的状态标签体系。

项目由一名维护者管理，GitHub Issue 的开关状态、内容和评论已经足以表达当前进度。不要仅为了流程完整而自动创建或应用以下标签：

- `needs-triage`
- `needs-info`
- `ready-for-agent`
- `ready-for-human`
- `wontfix`

现阶段可以按需使用 GitHub 已有的普通分类标签，但标签不作为自动化流程或任务状态的唯一依据。

如果未来出现以下情况，再重新评估是否引入正式的 triage 标签：

- 增加其他维护者或外部贡献者；
- 同时积压大量 Issue；
- 需要自动代理批量领取和执行任务；
- 需要明确区分等待信息、可自动执行和必须人工处理的任务。

在正式配置之前，任何技能或自动化代理都不得自行假定标签名称或创建一套新的任务状态。
